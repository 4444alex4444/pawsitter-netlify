PAWSITTER ДЛЯ NETLIFY
====================

Что внутри:
- красивый сайт PawSitter
- безопасная отправка заявок через Netlify Function
- токен Telegram больше не лежит в открытом коде
- анкета владельца и анкета ситтера обе отправляются в Telegram

КАК ЗАПУСТИТЬ НА NETLIFY
------------------------
1. Зайди в Netlify.
2. Создай новый site из этой папки или загрузи проект в Git и подключи его.
3. В настройках сайта открой:
   Site configuration -> Environment variables
4. Добавь 2 переменные:
   TELEGRAM_TOKEN = токен твоего Telegram-бота
   TELEGRAM_CHAT_ID = id чата, куда должны приходить заявки

ВАЖНО:
- scope переменных должен включать Functions
- не вставляй токен в код сайта

ОБЫЧНЫЕ BUILD НАСТРОЙКИ
-----------------------
Build command: npm run build
Publish directory: dist

ЕСЛИ NETLIFY САМ НЕ ПОДТЯНУЛ FUNCTIONS
-------------------------------------
Файл netlify.toml уже добавлен.
Обычно этого достаточно.

ЕСЛИ ХОЧЕШЬ ПРОСТО ЗАГРУЗИТЬ ПАПКУ
---------------------------------
Лучше загружать как проект с build, а не только готовую папку dist,
потому что функция submit-form должна тоже попасть в deploy.

КАК ПРОВЕРИТЬ TELEGRAM CHAT ID
------------------------------
1. Напиши что-нибудь своему боту в Telegram.
2. Убедись, что бот запущен и имеет доступ к чату.
3. Используй chat id этого чата.

ЧТО УЖЕ ИСПРАВЛЕНО
------------------
- убран открытый Telegram token из фронтенда
- убран пароль админки из браузера
- success-экран теперь показывается после реальной отправки
- ошибки отправки показываются пользователю
- анкета ситтера теперь тоже реально отправляется
